# Security Policy

At **Advans IT Solutions GmbH** we take the security of our software and
services seriously. This policy applies to all repositories in this
organization.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it to us
responsibly. **Do not** open a public issue, pull request, or discussion for
security problems.

Preferred reporting channels:

1. **GitHub Private Vulnerability Reporting** — use the **"Report a vulnerability"**
   button on the **Security** tab of the affected repository (when enabled).
2. **Email** — write to [info@advans.ch](mailto:info@advans.ch) with the subject
   line `SECURITY` and a clear description.

Please include, where possible:

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof of concept.
- The affected repository, version, or commit.
- Any suggested remediation.

## Our Commitment

- We will acknowledge your report within **5 business days**.
- We will keep you informed about the progress towards a fix.
- We will credit you for the discovery if you wish, once the issue is resolved.

We kindly ask that you give us a reasonable amount of time to address the issue
before any public disclosure.

## Scope

This policy covers the source code maintained in this organization. Security
issues regarding our products, services, or website
([advans.ch](https://advans.ch)) can be reported through the same channels.
